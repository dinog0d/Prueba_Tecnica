# Prueba_Tecnica
Este repositorio es creado con la finalidad de compartir mi evaluación de la prueba técnica para el perfil de practicante de fábrica.

<p>"- Ejercicio 01: Para realizar este ejercicio, me basé en una construcción clásica de la página de aprendizaje conocida como https://www.w3schools.com, una página que siempre ha sido un gran apoyo para aprender lógica básica de programación, la usé para encontrar sus plantillas de uso libre, de práctica y además para reforzar una buena práctica de estructuración de la página web. Así, construí el <"meta"> con la información de metadatos más relevante, la construcción del CSS de manera interna en el <"style"> y la estructura clásica del <"body">."
</p>

<p>"- Ejercicio 02: Para realizar este segundo ejercicio, me base en algunas prácticas encontradas en la misma página de https://www.w3schools.com, sumando ejercicios prácticos encontrados en https://lenguajejs.com/javascript/, esto con el fin de entender nuevamente los ejercicios básicos de JavaScript para el control en una prueba de HTML DOM. Utilicé el HTML del primer ejercicio para la base de la página y desde ahí realizar este ejercicio con las nuevas funcionalidades y estructura, la cual es: 

- Un segmento especializado en tener el input de la tarea y su botón correspondiente.
- Un segmento en el que se encontrarán las tareas.
- Cada tarea es un segmento en sí, que anida el nombre de la tarea y otro segmento que anida a su vez los botones de eliminar y de tarea completada."
</p>

<p>"- Ejercicio 03: Para realizar este tercer ejercicio, como se mencionan los datos específicos a usar y que el resultado se obtiene por consola, realicé un archivo .js en el cual cree dos funciones, una para encontrar los promedios de las notas y otro que haría la transformación de los datos usando la misma función de promedios, para finalmente aplicar esta función sobre los datos y usando console.log para mostrar los resultados de la transformación."
</p>

<p>"- Ejercicio 04: Para realizar este cuarto ejercicio, se tomó en cuenta primero que puede generarse diferentes errores por la dificultad de manejar este formato, por lo que primero al crear la función, se asegura que el formato sea el pedido en el ejercicio. Después de la validación del formato, se realiza la separación de los valores de hora y minuto, para entonces comprobar los condicionales para definir la hora, minuto y prefijo que serían retornados por la función."
</p>

<p>"- Ejercicio 05: Para realizar este quinto ejercicio, se tomaron los datos entregados y se observó la estructura en la cual estos están presentes, así se crea entonces la función que recibe estos datos y luego los posibles filtros que puede usarse para encontrar los datos buscados. La función entonces hace primero una validación de los datos, luego una validación de los filtros para finalmente retornar los datos que cumplan con las condiciones de filtrado."
</p>

<p>"- Ejercicio 06: Para realizar este sexto ejercicio, se creó el archivo JSON con el cual se va a trabajar, y en el archivo .js se crearon funciones asincrónicas para realizar el fetch de manera local del archivo y luego para realizar la búsqueda del producto a través del ID. Por lo tanto, esta segunda función realiza primero el fetch del listado usando la primera función, y luego hace una búsqueda por ID para encontrar el producto, levantando errores en caso de no encontrar el archivo JSON o en caso de ingresar un ID no valido."
</p>
